# 網路介面監控：單一欄位異常判讀

## 1. 單一欄位異常的定位

單一欄位異常主要回答：

```text
哪個指標不正常？
```

它可以用來做初步異常偵測與告警，但通常不能直接推論完整根因。真正的 RCA 需要搭配多欄位交互作用與時間窗關聯。

---

## 2. 常見單欄位指標解讀

| 指標大類 | 對應欄位 | 看什麼 | 異常時常見意義 |
|---|---|---|---|
| OCTETS | `INOCTETS`, `OUTOCTETS` | 流量大小 | 高流量、下載 / 上傳 / 備份 / 串流 / 業務流量增加 |
| PACKETS | `INUCASTPKTS`, `OUTUCASTPKTS`, `INNUCASTPKTS`, `OUTNUCASTPKTS` | 封包數量 | 大量連線、小封包、掃描、服務請求增加 |
| ERRORS | `INERRORS`, `OUTERRORS` | 傳輸品質 | 線材、port、NIC、SFP、duplex / speed mismatch |
| DISCARDS | `INDISCARDS`, `OUTDISCARDS` | 丟棄 / queue | queue 滿、buffer 不足、QoS 丟棄、壅塞 |
| BROADCAST | `INBROADCASTPKTS`, `OUTBROADCASTPKTS` | Broadcast 行為 | ARP storm、DHCP storm、broadcast storm、switch loop |
| MULTICAST | `INMULTICASTPKTS`, `OUTMULTICASTPKTS` | Multicast 行為 | IGMP 問題、multicast flooding、IPTV / routing 異常 |
| UNKNOWNPROTOS | `INUNKNOWNPROTOS` | 未知協定 | 非標準協定、掃描、設定錯誤、新設備 / 新服務 |

---

## 3. 常見時間序列 Pattern

常見 pattern 包含：

1. 持續偏高：長時間維持高水準。
2. 突然尖峰：短時間突然上升後回落。
3. 週期性波動：固定時間重複出現高峰。
4. 緩慢上升：長期趨勢逐漸變高。
5. 稀疏非零：平常為 0，偶爾出現非零值。
6. 反覆 spike：不規則地反覆出現尖峰。

---

## 4. 不同欄位適合觀察的 Pattern

不是所有欄位都同樣常出現四種 pattern。欄位特性不同，解讀也不同。

| 指標類型 | 持續高平台 | 突然尖峰 | 週期性波動 | 緩慢上升 | 備註 |
|---|---:|---:|---:|---:|---|
| OCTETS | 常見 | 常見 | 常見 | 常見 | 適合 baseline、forecast、capacity planning |
| PACKETS | 常見 | 常見 | 常見 | 常見 | 可搭配 OCTETS 判斷小封包或大檔傳輸 |
| ERRORS | 較少 | 常見 | 可能 | 可能但較少 | 通常是稀疏型，重點看非零、spike、反覆出現 |
| DISCARDS | 常見 | 常見 | 可能 | 可能 | 常與 queue、buffer、壅塞相關 |
| BROADCAST | 可能 | 常見 | 常見 | 可能 | 重點看 storm、週期性 discovery、多 port 同步 |
| MULTICAST | 常見 | 可能 | 常見 | 可能 | 重點看 flooding、IGMP、routing / streaming |
| UNKNOWNPROTOS | 較少 | 常見 | 可能 | 較少 | 通常應接近 0，重點看出現、反覆與來源 |

---

## 5. OCTETS 單欄位異常

### 常見 pattern

| Pattern | 可能意義 |
|---|---|
| 持續高平台 | 長時間高流量、備份、影音串流、尖峰使用 |
| 突然尖峰 | 突發下載、短時間大量傳輸 |
| 週期性波動 | 固定批次作業、上下班流量、定期同步 |
| 緩慢上升 | 使用量成長、容量壓力上升 |

### 注意

`OCTETS` 高不一定是故障，可能只是正常業務量增加。需要搭配 `DISCARDS`、`ERRORS`、`PACKETS` 判斷是否真的異常。

---

## 6. PACKETS 單欄位異常

### 常見 pattern

| Pattern | 可能意義 |
|---|---|
| 持續高平台 | 連線數長時間偏高 |
| 突然尖峰 | 大量連線、掃描、請求暴增 |
| 週期性波動 | heartbeat、service discovery、批次請求 |
| 緩慢上升 | 使用者數、連線數或服務請求量逐漸增加 |

### 注意

`PACKETS` 高但 `OCTETS` 不高，通常代表大量小封包，可能是掃描、heartbeat、短連線、遊戲或即時通訊封包。

---

## 7. ERRORS 單欄位異常

`INERRORS` / `OUTERRORS` 通常是稀疏型指標。健康網路中常常長期為 0。

### 常見 pattern

| Pattern | 可能意義 |
|---|---|
| 長期為 0 | 正常 |
| 偶發 spike | 短暫干擾或偶發錯誤 |
| 反覆 spike | 間歇性線路、port、SFP 或 NIC 問題 |
| 持續上升 | 明顯實體層或鏈路層異常 |

### 常見原因

- 網路線品質差或接頭鬆動
- SFP / 光模組異常
- NIC 或 switch port 問題
- Duplex mismatch
- Speed negotiation 問題
- CRC / frame / alignment errors

---

## 8. DISCARDS 單欄位異常

`DISCARDS` 代表封包被丟棄，但封包本身不一定壞。通常反映 queue、buffer、QoS 或壅塞問題。

### 常見 pattern

| Pattern | 可能意義 |
|---|---|
| 短暫 spike | 突發流量造成短暫丟棄 |
| 持續高平台 | 長時間 queue 滿或壅塞 |
| 週期性丟棄 | 固定批次流量造成 queue 壓力 |
| 緩慢上升 | 容量不足逐漸惡化 |

### 與 ERRORS 差異

| 指標 | 意義 | 常見問題 |
|---|---|---|
| ERRORS | 封包傳輸品質有問題 | 線材、port、NIC、SFP |
| DISCARDS | 封包未壞，但設備主動丟掉 | queue、buffer、QoS、壅塞 |

---

## 9. BROADCAST / MULTICAST 單欄位異常

### Broadcast

Broadcast 高可能代表：

- ARP storm
- DHCP storm
- Broadcast storm
- Switch loop
- STP 問題
- VLAN 設計過大

### Multicast

Multicast 高可能代表：

- IGMP snooping 問題
- Multicast flooding
- IPTV / video streaming 異常
- Routing protocol 異常
- mDNS / SSDP 增加

---

## 10. UNKNOWNPROTOS 單欄位異常

`INUNKNOWNPROTOS` 通常應接近 0。

異常時可能代表：

- 非標準協定
- 掃描或攻擊探測
- 新設備 / 新服務導入
- VLAN / routing 設定錯誤
- 設備不支援某種協定

---

## 11. 單欄位異常的限制

單欄位異常適合做初步告警，但有明顯限制：

- `OCTETS` 高不一定是壅塞。
- `PACKETS` 高不一定是攻擊。
- `ERRORS` 高才比較接近傳輸品質問題。
- `DISCARDS` 高不一定是線路壞，可能是 queue 滿。
- `BROADCAST` 高需要看是否多 port 同步。
- `MULTICAST` 高需要看是否符合應用情境。

因此 RCA 需要多欄位交互作用。

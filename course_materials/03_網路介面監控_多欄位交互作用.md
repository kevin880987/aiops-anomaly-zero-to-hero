# 網路介面監控：多欄位交互作用與問題判讀

## 1. 為什麼需要多欄位交互作用？

單一欄位異常只能回答：

```text
哪個指標異常？
```

多欄位交互作用可以進一步回答：

```text
這是哪一種異常型態？
可能原因是什麼？
應該往哪個方向排查？
```

這正是 AIOps / RCA 的核心：

```text
多欄位異常組合 → 問題型態 → 根因候選 → 建議處置
```

---

## 2. 建議衍生指標

原始欄位可先整理成高階特徵：

```text
traffic = INOCTETS + OUTOCTETS
packets = INUCASTPKTS + OUTUCASTPKTS
errors = INERRORS + OUTERRORS
discards = INDISCARDS + OUTDISCARDS
broadcast = INBROADCASTPKTS + OUTBROADCASTPKTS
multicast = INMULTICASTPKTS + OUTMULTICASTPKTS
non_unicast = INNUCASTPKTS + OUTNUCASTPKTS
unknown = INUNKNOWNPROTOS
```

再衍生比例型指標：

```text
avg_packet_size = traffic / packets
discard_rate = discards / packets
error_rate = errors / packets
broadcast_ratio = broadcast / packets
multicast_ratio = multicast / packets
in_out_ratio = INOCTETS / OUTOCTETS
```

---

## 3. 多欄位交互作用總表

| 問題 | 欄位交互作用 | 典型時間序列解讀 |
|---|---|---|
| 正常業務流量增加 | `OCTETS ↑` + `UCASTPKTS ↑` + `DISCARDS ≈ 0` | 流量與封包同步上升，但無丟棄 |
| 大量小封包 / 掃描 | `UCASTPKTS ↑↑` + `OCTETS 小幅↑` | 封包數暴增，流量不一定大 |
| 大檔案傳輸 | `OCTETS ↑↑` + `UCASTPKTS 中度↑` | 平均封包較大 |
| 頻寬壅塞 | `OCTETS ↑↑` + `DISCARDS ↑` | 高流量後出現丟棄 |
| Queue 滿 / Bufferbloat | `DISCARDS ↑` + `ERRORS ≈ 0` | 主動丟棄，不是線路錯誤 |
| 線路品質問題 | `ERRORS ↑` + `DISCARDS 不一定↑` | CRC / frame / port 問題 |
| 高負載下線路不穩 | `OCTETS ↑` 後 `ERRORS ↑` | 流量高時才出錯 |
| Broadcast storm | `BROADCAST ↑↑` + `NUCAST ↑↑` + 多 port 同時↑ | L2 loop / ARP storm |
| Multicast flooding | `MULTICAST ↑↑` + 多 port 同時↑ | IGMP / multicast 設定問題 |
| 異常設備大量發送 | 多個 `OUT*PKTS ↑` + `OUTOCTETS ↑` | 某設備發送異常 |
| Unknown protocol | `INUNKNOWNPROTOS ↑` + `INPKTS ↑` | 非標準協定 / 掃描 / 設定錯誤 |

---

## 4. 正常業務流量增加

### 欄位交互作用

```text
OCTETS ↑
+ UCASTPKTS ↑
+ DISCARDS ≈ 0
+ ERRORS ≈ 0
```

### 時間序列樣態

```text
OCTETS      ____/^^^^^^^^^^^^^
UCASTPKTS   ____/^^^^^^^^^^^^^
DISCARDS    __________________
ERRORS      __________________
```

### 解讀

流量與封包同步上升，但沒有丟棄或錯誤，通常是正常業務量增加，而不一定是故障。

### 可能原因

- 使用者數增加
- 正常服務流量增加
- 批次作業
- 資料同步
- 影音串流

---

## 5. 大量小封包 / 掃描 / 短連線暴增

### 欄位交互作用

```text
UCASTPKTS ↑↑
+ OCTETS 小幅 ↑
+ avg_packet_size ↓
```

### 時間序列樣態

```text
UCASTPKTS   ____/^^^^^^^^^^^^^
OCTETS      ____/---小幅上升---
```

### 解讀

封包數暴增，但總流量不一定大，代表平均封包較小。

### 可能原因

- Port scan
- 小封包 DDoS
- 大量 TCP SYN
- 大量短連線
- Heartbeat / keep-alive 暴增
- 遊戲或即時通訊封包

---

## 6. 大檔案傳輸 / 備份 / 串流

### 欄位交互作用

```text
OCTETS ↑↑
+ UCASTPKTS 中度 ↑
+ avg_packet_size ↑
```

### 時間序列樣態

```text
OCTETS      ____/^^^^^^^^^^^^^
UCASTPKTS   ____/---中度上升---
```

### 解讀

總資料量很大，但封包數沒有同比例暴增，表示平均封包較大。

### 可能原因

- 備份
- 檔案同步
- 影音串流
- 大檔下載
- VM image / log archive 傳輸

---

## 7. 頻寬壅塞 / Queue 滿

### 欄位交互作用

```text
OCTETS ↑↑
+ DISCARDS ↑
+ ERRORS ≈ 0
```

### 時間序列樣態

```text
OCTETS      ____/^^^^^^^^^^^^^
DISCARDS    ________/^^^^^
ERRORS      __________________
```

### 解讀

高流量後出現丟棄，但沒有 errors，通常不是線路壞，而是 queue / buffer / 頻寬壓力。

### 可能原因

- 頻寬不足
- Queue / buffer 滿
- QoS policy 丟棄
- Router / switch 處理能力不足
- 突發流量超過設備處理能力

---

## 8. 線路品質問題

### 欄位交互作用

```text
ERRORS ↑
+ DISCARDS 不一定 ↑
```

### 時間序列樣態

```text
ERRORS      __▲____▲______▲____
DISCARDS    ___________________
```

### 解讀

Errors 反覆 spike，但 discards 不一定高，通常比較像線路品質或實體層問題。

### 可能原因

- 網路線鬆動
- 接頭不良
- 光模組 / SFP 問題
- NIC 異常
- Switch port 異常
- Duplex / speed mismatch

---

## 9. 高負載下線路不穩

### 欄位交互作用

```text
OCTETS ↑
後 ERRORS ↑
```

### 時間序列樣態

```text
OCTETS      ____/^^^^^^^^^^^^^
ERRORS      ________/^^^^^
```

### 解讀

平常沒事，但流量高時 errors 才出現。這代表線材、port 或設備品質可能在高負載下不穩。

---

## 10. Broadcast Storm / L2 Loop

### 欄位交互作用

```text
BROADCAST ↑↑
+ NUCAST ↑↑
+ 多 port 同時 ↑
+ DISCARDS 可能 ↑
```

### 時間序列樣態

```text
Port A BROADCAST   ________/^^^^^^^^^
Port B BROADCAST   ________/^^^^^^^^^
Port C BROADCAST   ________/^^^^^^^^^
```

### 解讀

Broadcast 或 non-unicast 在多個 port 同時暴增，通常是二層網路問題。

### 可能原因

- Switch loop
- STP 問題
- ARP storm
- DHCP storm
- VLAN 設計過大
- 某設備大量 broadcast

---

## 11. Multicast Flooding / IGMP 問題

### 欄位交互作用

```text
MULTICAST ↑↑
+ 多個無關 port 同時 ↑
+ DISCARDS 可能 ↑
```

### 時間序列樣態

```text
Port A MULTICAST   ________/^^^^^^^
Port B MULTICAST   ________/^^^^^^^
Port C MULTICAST   ________/^^^^^^^
```

### 解讀

Multicast 流量出現在不該收到的 port，或多個無關 port 同時上升，可能是 multicast flooding。

### 可能原因

- IGMP snooping 未啟用
- IGMP querier 問題
- IPTV / 串流異常
- Routing protocol multicast 異常

---

## 12. 異常設備大量發送

### 欄位交互作用

```text
OUTUCASTPKTS ↑
+ OUTBROADCASTPKTS ↑ 或 OUTMULTICASTPKTS ↑
+ OUTOCTETS ↑
+ 集中在某個 port
```

### 解讀

某個 port 的多個 outbound 指標同步上升，代表該 port 連接的設備可能是 root cause candidate。

### 可能原因

- 某台 server 異常
- Malware / 中毒
- 備份程序異常
- Service discovery 暴增
- 掃描或攻擊行為
- 設定錯誤

---

## 13. Unknown Protocol

### 欄位交互作用

```text
INUNKNOWNPROTOS ↑
+ INPKTS ↑
```

### 解讀

設備收到無法辨識的協定封包，且同時 inbound packet 增加。

### 可能原因

- 非標準協定
- 掃描或攻擊探測
- 新設備 / 新服務導入
- VLAN / routing 設定錯誤
- 設備不支援某協定

---

## 14. 快速判斷規則

```text
OCTETS 高 + DISCARDS 高 + ERRORS 低
→ 壅塞 / queue 問題

ERRORS 高 + DISCARDS 低
→ 線路 / port 品質問題

PACKETS 高 + OCTETS 不高
→ 小封包 / 掃描 / 短連線

BROADCAST 高 + 多 port 同時高
→ Broadcast storm / L2 loop

MULTICAST 高 + 多 port 同時高
→ Multicast flooding / IGMP 問題

INUNKNOWNPROTOS 高 + INPKTS 高
→ 協定異常 / 掃描 / 設定錯誤
```
